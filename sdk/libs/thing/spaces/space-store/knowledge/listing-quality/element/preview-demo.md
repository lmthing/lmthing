# Preview & Demo

Show, don't tell. A preview or demo lets potential users see the agent in action before committing to a download or purchase. Good previews can double conversion rates compared to description-only listings.

## Preview Options

- **Screenshot** — A sample conversation showing the agent's best response
- **Recorded demo** — A short walkthrough of a typical interaction
- **Live preview** — For API-access agents, let users try a limited version
- **Curated output** — A polished example of what the agent produces

## What to Showcase

Focus on the agent's "wow moment" — the capability that makes users say "I need this":

- For knowledge-heavy agents, show how the agent uses its knowledge to give uniquely valuable responses that a generic LLM couldn't match
- For tool-using agents, show a complex task being completed step by step
- For multi-agent spaces, show the coordination between specialists
- For workflow agents, show the end-to-end flow from start to finished output

## Best Practices

- Keep demos under 2 minutes — attention spans are short
- Use realistic scenarios, not contrived examples
- Show both the input (what the user asked) and the output (what the agent produced)
- Include edge cases that demonstrate the agent's robustness
- Update previews when you improve the agent

## Impact

Listings with previews consistently outperform those without. Users are more confident in their purchase decision when they can see exactly what they're getting. This is especially important for paid agents where the preview reduces purchase anxiety.
