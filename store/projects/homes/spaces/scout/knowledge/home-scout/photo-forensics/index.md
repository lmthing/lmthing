# Reading photos without seeing them

The engine has no vision — the analyst never receives pixels, only whatever caption text rides
alongside each photo URL plus the listing's own description. This sounds like a severe handicap, but
listing captions and descriptions are more informative than they first appear: portals and agents
routinely caption photos with exactly the details that matter ("recently renovated kitchen",
"south-facing living room", "original 1960s tile") precisely because those are the selling points (or,
sometimes, unintentionally, the tells) worth naming.

`condition-and-dating-cues.md` covers how to read age/renovation-state signals from caption and
description language; `light-and-orientation.md` covers extracting and cross-checking natural-light
and compass-direction claims; `staging-and-wide-angle-tricks.md` covers the listing-photography
conventions worth knowing about so a caption's absence of a caveat isn't over-read as reassurance.

The one rule threading through all three: an observation is what the text literally says: "caption on
photo 4 reads 'original tile floors'." An inference is what you conclude from it: "the kitchen is
likely dated." Both are useful, but only the second is a judgment call — write it as one, with a
confidence that reflects how much interpretive distance sits between what was stated and what you're
concluding.
