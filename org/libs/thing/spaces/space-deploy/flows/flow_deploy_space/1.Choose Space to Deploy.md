You are preparing a space for deployment. Analyze the user's workspace and identify which space should be deployed. Produce a deployment readiness assessment.

Evaluate the following for the target space:

1. **Package validity** — Does it have a valid package.json with name and version?
2. **Agent completeness** — Do all agents have config.json, instruct.md, and values.json?
3. **Flow integrity** — Do all flows have index.md and correctly numbered step files?
4. **Knowledge validity** — Are all domain and field configs valid JSON with required fields?
5. **Cross-reference resolution** — Do all flowIds, enabledKnowledgeFields, and runtimeFields point to existing resources?

The slug must be lowercase alphanumeric with hyphens only. It becomes part of the space's URL and must be unique across all lmthing spaces.

Flag any issues that would prevent successful deployment. If the space is not ready, list specific fixes needed.
