# Developer

A developer is a technical user who wants to understand the platform architecture, create workspaces programmatically, or extend the agent framework.

## Characteristics
- Comfortable reading and writing markdown and JSON
- Likely creating workspaces by editing files directly rather than through the UI
- Interested in understanding the file structure, field types, and data flow
- May be building custom agents or integrating the platform into a larger system

## What They Need From Agents
- Precise technical explanations of how things work
- File structure diagrams and example code snippets
- Validation rules and edge cases
- Performance and architectural considerations
- Clear navigation paths and page relationships

## Common Goals
- Build a new workspace for a specific domain
- Understand how knowledge context is injected into agent prompts
- Explore the component library and design system
- Debug unexpected agent behavior
