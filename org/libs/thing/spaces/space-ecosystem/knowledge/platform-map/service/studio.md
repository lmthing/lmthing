# lmthing.studio

The agent builder UI and primary development surface for the lmthing platform. Studio is where you design, build, test, and iterate on AI agents.

## What It Does

Studio provides a visual workspace for creating agents, flows, and knowledge bases. You can write system prompts, configure tools, wire knowledge domains, and test conversations — all in one interface. The workspace is backed by a virtual file system (VFS) that syncs to your private GitHub repo.

## Key Features

- **Agent editor** — Visual builder for creating agents with system prompts, tool configuration, and knowledge wiring
- **Knowledge designer** — Create and manage knowledge domains, fields, and option content
- **Flow builder** — Design step-by-step workflows that agents execute via slash actions
- **Live conversations** — Test agents in real-time with streaming responses
- **Workspace management** — File tree, raw editor, and GitHub sync (push/pull)
- **Environment configuration** — Manage API keys, environment variables, and model settings

## When to Use

Start here if you're building or customizing agents. Studio is the hub for all creative work — designing agent personas, structuring knowledge, and crafting workflows. Most users spend the majority of their time in Studio.

## How It Connects

Agents built in Studio can be deployed to containers via Space, published to the Store, or run on your personal Computer node. Workspace data syncs through GitHub for version control and persistence.
