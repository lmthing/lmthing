# Take Action

Based on your findings, recommend and execute appropriate actions:

**For unhealthy running spaces:**
- Restart: stop-space then start-space (preserves data, fresh machine boot)
- If restart doesn't help, check for application-level issues

**For idle running spaces:**
- Stop to save costs: stop-space (volume preserved, no billing)
- Recommend a schedule: run during work hours, stop overnight

**For failed spaces:**
- Try reprovisioning in a different region
- Check configuration for errors
- Review logs for the specific failure cause

**For long-stopped spaces:**
- Ask: is this still needed? If not, delete to clean up resources
- If needed occasionally, keep stopped — no ongoing cost

**Destructive actions (delete):**
- Always confirm with the user before deleting
- Explain that deletion destroys the machine, volume, and app permanently
- Recommend pushing any important data to GitHub first
- Status changes to "destroyed" (soft delete in database)

Document what actions were taken and verify each one resolved the intended issue.
