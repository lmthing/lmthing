# Write Option Files

For each select/multiSelect field, write the option markdown files.

**Each option file contains:**

### YAML Frontmatter
```yaml
---
title: Option Title
description: One-line summary
order: 1
---
```

### Body (150-300 words)
Write content that would meaningfully change how an agent behaves when this option is selected. Structure with:

1. **Definition** — What this option represents
2. **Characteristics** — Key traits or properties
3. **Agent guidance** — How the agent should adapt when this option is active
4. **Examples** (optional) — Concrete illustrations

### Quality Standards

**Good option content:**
- Actionable — selecting it changes agent behavior
- Distinct — noticeably different from sibling options
- Self-contained — makes sense without reading other options
- Substantive — at least 150 words of meaningful guidance

**Weak option content:**
- Just a definition with no behavioral guidance
- Duplicates information from sibling options
- Too short to meaningfully influence the agent
- Generic advice that applies regardless of the selection

### File Naming
- Filename: `{slug}.md` in kebab-case
- The slug must match what's referenced as the `default` value in the field's `config.json`
- Order frontmatter values sequentially (1, 2, 3...) to control display order

Generate all option files. For large spaces, generate the first domain completely as a template, then let the user continue the pattern.
