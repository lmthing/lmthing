For every select and multiSelect field defined in the previous step, generate the complete set of option markdown files.

Each option file must contain:

**YAML frontmatter:**
- `title` — Human-readable option title
- `description` — One-line summary of the option
- `order` — Sequential integer (1, 2, 3...) controlling display order

**Body content (150-300 words)** structured as:
1. **Definition** — What this option represents in clear terms
2. **Characteristics** — Key traits, behaviors, or properties that distinguish it
3. **Agent guidance** — Specific instructions for how the agent should adapt its behavior when this option is selected
4. **Examples** (optional) — Concrete illustrations of this option in practice

Apply these quality criteria to every option:
- **Actionable** — Selecting this option must produce a meaningful change in agent behavior
- **Distinct** — Must be noticeably different from sibling options in the same field
- **Self-contained** — Must make sense without reading other options
- **Substantive** — At least 150 words of meaningful guidance

File naming: each option uses its kebab-case slug as the filename (e.g., `beginner.md`). The slug must match what is referenced as the `default` value in the field's config.json.

Generate all option files grouped by domain and field. Use the space brief and domain plan to ensure content is specific to this space's subject matter, not generic.
