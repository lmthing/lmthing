Create the flows referenced by agents' slash actions and validate the entire space for correctness.

For each flow referenced in an agent's `flowId`:

1. Generate the flow folder structure: `flows/flow_{action_name}/`
2. Generate `index.md` with frontmatter (name, description, tags) and numbered step links
3. Generate each step file following the standard step format with frontmatter (description, type, model, temperature) and LLM prompt content with output schemas
4. Design 4-8 steps per flow following a logical progression (typically Gather, Plan, Execute, Review)

Then run a complete cross-reference validation across the entire space:

- All `flowId` values in agent instruct.md files match a flow folder name
- All `enabledKnowledgeFields` use `domain-` prefix and match knowledge folder names
- All field `default` values match an option file slug
- All `runtimeFields` domain keys match knowledge domain folders
- All `runtimeFields` field values match field folders within those domains
- All JSON files are valid with no trailing commas
- All YAML frontmatter is valid
- Naming conventions are correct: folders in kebab-case, agents in PascalCase, flows in `flow_` + snake_case, variables in camelCase

Report any issues found and provide the corrected values.
