# Wire Flows and Test

Create flows for agent slash actions and verify everything works together.

**Goal:** Build the flows referenced by agents and validate the complete space.

### Create Flows

For each flow referenced in an agent's `flowId`:

1. Create the folder: `flows/flow_{action_name}/`
2. Write `index.md` with title, description, and numbered step links
3. Write each step file (`1.Step Name.md`, `2.Step Name.md`, etc.)

Follow flow design patterns:

- **4-8 steps** per flow
- Steps should follow Gather → Plan → Execute → Review progression
- Each step should produce a concrete output or decision

### Validation Checklist

Run through these cross-reference checks:

- [ ] All `flowId` values in agent instruct.md match a flow folder name
- [ ] All `enabledKnowledgeFields` use `domain-` prefix and match knowledge folder names
- [ ] All field `default` values match an option file slug
- [ ] All `runtimeFields` domain keys match knowledge domain folders
- [ ] All `runtimeFields` field values match field folders within those domains
- [ ] All JSON files are valid (no trailing commas)
- [ ] All YAML frontmatter is valid
- [ ] Folders are kebab-case, agents are PascalCase, flows are `flow_` + snake_case, variables are camelCase

### Test

Start a conversation with each agent and verify:

- Runtime fields are prompted before the conversation begins
- Slash commands appear and trigger the correct flow
- Knowledge context is injected and shapes the agent's responses
