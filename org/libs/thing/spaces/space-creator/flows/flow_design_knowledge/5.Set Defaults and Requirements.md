Finalize the configuration for every field by setting default values and requirement flags. Generate the complete `config.json` for each field.

For each field, determine:

**Default value:**
- For `select` fields: the slug of the most common or safest option. Must exactly match an option file's slug (kebab-case filename without `.md`).
- For `multiSelect` fields: an array of default slugs, or empty array.
- For `text` fields: empty string `""`.
- For `number` fields: a sensible starting value.

**Required flag:**
Set `required: true` ONLY when:
- The agent's behavior is fundamentally different based on the selection
- There is no safe default that covers most users
- Using the wrong value would produce unhelpful or misleading output

Set `required: false` (preferred for most fields) when:
- A sensible default covers 80% of users
- The field provides nice-to-have context rather than essential configuration
- Asking too many questions before starting would create friction

Generate the complete config.json for each field:
```json
{
  "label": "Field Label",
  "description": "What this field captures",
  "fieldType": "select",
  "required": false,
  "default": "option-slug",
  "variableName": "fieldVariable",
  "renderAs": "field"
}
```

Validate that every `default` value matches an actual option file slug in that field's folder. Report any mismatches.
