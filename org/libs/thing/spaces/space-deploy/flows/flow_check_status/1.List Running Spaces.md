# List Running Spaces

Use list-spaces to get all non-destroyed spaces and review their status:

For each space, note:
- **Name and slug** — Which space is this?
- **Status** — created, provisioning, running, stopped, or failed
- **Region** — Where is it deployed?
- **Created/updated timestamps** — How old is this deployment?

**Triage by status:**
- **Running** — Active and costing money. Is it needed? Check health.
- **Failed** — Broken. Needs investigation or cleanup.
- **Stopped** — Preserved but inactive. Should it be restarted, kept, or deleted?
- **Provisioning** — Still setting up. Check if it's stuck (>2 minutes).

Identify spaces that need attention: failed deployments that should be investigated, running spaces that might be idle, or stopped spaces that have been inactive for too long.
