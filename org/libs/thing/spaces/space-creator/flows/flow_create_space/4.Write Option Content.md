# Write Option Content

For each select/multiSelect field from Step 3, write the option markdown files.

**Goal:** Create rich, actionable option content that meaningfully changes agent behavior.

Each option file needs:

### YAML Frontmatter
```yaml
---
title: Option Title
description: One-line summary of this option
order: 1
---
```

### Body Content (150-300 words)
Structure each option with:
1. **What it is** — A clear definition or description
2. **Characteristics** — Key traits, behaviors, or properties
3. **How to adapt** — Specific guidance for the agent when this option is selected
4. **Examples** (optional) — Concrete illustrations

### Quality Checklist
- Is the content actionable? (Would selecting this option change the agent's behavior?)
- Is it distinct from other options? (Would a different selection produce noticeably different results?)
- Is it self-contained? (Does it make sense without reading the other options?)
- Does the filename match the slug in kebab-case?

For beginners, generate all option files in full. For intermediate/advanced users, generate 1-2 examples and let them complete the rest.
