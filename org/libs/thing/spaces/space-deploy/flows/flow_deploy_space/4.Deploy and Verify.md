You are executing the deployment of a space to Fly.io. Using the deployment readiness, region config, and access config from previous steps, produce a deployment plan and verification checklist.

The deployment process:
1. Call create-space with `{ name, slug, description, region }`
2. The API returns immediately with status "provisioning"
3. Provisioning creates: Fly.io app, 1 GB volume, machine, and health checks
4. Expect 30-60 seconds for provisioning to complete
5. Poll via get-space until status transitions from "provisioning"

After provisioning completes, apply any access control settings via update-space (auth_enabled, custom_domain, app_config).

Generate a verification checklist tailored to this specific deployment's configuration. Include all applicable checks based on whether auth is enabled, a custom domain is set, etc.

If provisioning fails, recommend: check the status for error details, try a different region if capacity-related, verify the slug is valid.
