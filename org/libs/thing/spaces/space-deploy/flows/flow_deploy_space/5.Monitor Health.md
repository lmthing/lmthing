You are setting up ongoing monitoring for a newly deployed space. Based on the deployment configuration and the space's purpose, generate a monitoring plan with cost management recommendations.

Consider the following monitoring areas:

1. **Health checks** — HTTP GET to /health every 10 seconds, 5-second timeout. Healthy spaces respond with 200 status consistently.
2. **Status monitoring** — Periodic checks via list-spaces or get-space for status changes from "running" to "failed".
3. **Performance tracking** — Response time monitoring, resource usage patterns.
4. **Cost management** — Running spaces accrue compute costs continuously. Recommend start/stop schedules based on usage patterns.

Generate recommendations tailored to this space's characteristics:
- Always-on production spaces need different monitoring than development or demo spaces
- High-traffic spaces need performance monitoring emphasis
- Cost-sensitive deployments need aggressive stop/start scheduling

Include clear action triggers: when to restart, when to investigate, when to stop, when to delete.
