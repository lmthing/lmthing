# Configure Runtime Fields

Runtime fields are knowledge fields whose values are not pre-filled — they prompt the user for input when starting a conversation.

Configure them in each agent's `config.json` under `runtimeFields`:

```json
{
  "runtimeFields": {
    "domain-name": ["field-slug", "another-field-slug"]
  }
}
```

Use runtime fields for:

- **User-specific values** — name, company, role
- **Session-specific choices** — task goal, target audience
- **Dynamic configurations** — anything that changes per conversation

**Examples:**

- `"user-context": ["name", "role"]` — prompts for user's name and role before each chat
- `"content-type": ["tone"]` — lets the user pick communication tone per session

**Best practices:**

- Mark frequently-changing values as runtime fields
- Default values in `config.json` reduce friction for optional fields
- Required fields (`required: true`) will block the conversation until filled
