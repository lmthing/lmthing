Read the agent's instruct.md and analyze it against the problem diagnosis from the previous step. Evaluate each aspect of the prompt systematically.

Analyze:
- **Prompt strategy** — is there a clear strategy (identity-framing, constraint-setting, knowledge-referencing)? Or is the prompt unstructured?
- **Role clarity** — is the agent's identity and expertise clearly defined? Vague roles produce vague responses.
- **Boundaries** — is there a "do NOT" section? Missing boundaries cause role drift.
- **Knowledge wiring** — are enabledKnowledgeFields appropriate? Too many bloats context; too few leaves gaps.
- **Tool guidance** — if tools are enabled, does the prompt explain when and how to use them?
- **Model match** — is the selected model suited to the prompt style? Claude excels at nuanced instructions, GPT-4o at tool calling, Gemini at long context.
- **Temperature** — is it appropriate for the task? Lower (0.2-0.4) for precision, higher (0.6-0.8) for creativity.

Identify the root cause connecting the diagnosed symptoms to specific prompt deficiencies. The problem is usually structural (wrong strategy, missing boundaries) rather than cosmetic (word choice).
