Call the provision-computer endpoint with the chosen region to create the machine. Track the asynchronous provisioning process:

1. The API returns immediately with status "provisioning"
2. In the background: Fly.io app is created, volume is allocated, machine is launched
3. Health checks verify the machine is responsive
4. Status updates to "running" on success or "failed" on error

Provisioning typically takes 30-60 seconds. If status remains "provisioning" beyond 2 minutes, flag a potential issue. If it fails, recommend trying a different region or rechecking subscription status.

One computer per user is enforced — if one already exists, generate instructions to start it instead.
