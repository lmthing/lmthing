# Sync With GitHub

Ensure your workspace is in sync with your private GitHub repository:

- **Pull first** — Download the latest state from GitHub before making local changes. This prevents conflicts with changes made elsewhere (other devices, collaborators).
- **Review changes** — Check what's different between your local VFS and the remote repo. Identify new files, modified files, and deletions.
- **Resolve conflicts** — If both local and remote have changes to the same file, resolve them. Choose the correct version or merge the changes manually.
- **Push when ready** — Upload your changes to GitHub. This is the only way to persist workspace data long-term — the VFS is ephemeral.

Push frequently to avoid losing work. The VFS resets when you close the browser or start a new session.
