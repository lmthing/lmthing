# Update Subscription

Guide the user through subscription changes:

**New subscription** — Use the create-checkout endpoint with the appropriate price_id. This creates a Stripe Checkout session and returns a URL. The user completes payment on Stripe's hosted page and is redirected back to lmthing.

**Manage existing** — Use the billing-portal endpoint to open Stripe's Customer Portal. Here users can update payment methods, switch plans, cancel subscriptions, and view invoices.

**Important behaviors:**
- Upgrading takes effect immediately — new models and limits are available right away
- Downgrading takes effect at the end of the current billing period
- Canceling the Computer tier automatically destroys the Fly.io machine (data is lost)
- Stripe creates the customer record lazily on first AI call or checkout

Confirm the user understands what changes before proceeding, especially for downgrades that may affect running services.
