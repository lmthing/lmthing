# Non-technical User

A non-technical user builds and manages workspaces entirely through the Studio interface — no file editing required.

## Characteristics
- Unfamiliar with JSON, YAML, or markdown syntax
- Builds knowledge areas and agents through point-and-click UI
- Relies heavily on modals, form inputs, and visual feedback
- Benefits from clear step-by-step guidance and tooltips

## What They Need From Agents
- Plain-language explanations avoiding technical jargon
- Step-by-step instructions that reference visible UI elements
- Reassurance about what each action does before they do it
- Guidance on when to click which buttons and what each modal is for

## Common Goals
- Create a new workspace for their team's use case
- Configure an agent without writing code
- Add and organize knowledge content
- Run and test an agent in the sandbox chat
