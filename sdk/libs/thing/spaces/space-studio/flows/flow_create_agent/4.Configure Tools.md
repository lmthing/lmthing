Using the agent definition, knowledge configuration, and instruct.md content from previous steps, generate the agent's config.json and values.json files.

For config.json:
- Set runtimeFields mapping domain names to field slugs that must be populated before the agent runs
- Only include fields the agent genuinely cannot function without — keep this minimal
- Every domain and field slug referenced must exist in the space's knowledge/ directory

For values.json:
- Generate an empty object `{}` — this file is populated at runtime with user selections

Validate cross-references:
- Every flowId in slash action tags must match an existing flow directory name (flow_snake_case)
- Every enabledKnowledgeFields entry must reference an existing domain (domain-{folder-name})
- Every runtimeFields entry must point to valid domain and field combinations
