Analyze the current sync state between the local VFS and the user's private GitHub repository, then produce a sync action plan.

Determine the current state:
- Has the user pulled the latest remote state? If not, recommend pulling first to prevent conflicts.
- Identify files that are new locally (not yet pushed), modified locally, or deleted locally.
- Identify files that are new remotely, modified remotely, or deleted remotely.
- Detect conflicts where both local and remote have changes to the same file.

For each conflict, recommend a resolution strategy:
- If the local version is a clear improvement, recommend keeping local
- If the remote version is newer and local has no meaningful changes, recommend keeping remote
- If both versions have valuable changes, recommend manual merge

Remind the user that the VFS is ephemeral — pushing to GitHub is the only way to persist workspace data long-term.
