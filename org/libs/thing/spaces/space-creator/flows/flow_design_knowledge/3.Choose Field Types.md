# Choose Field Types

For each field from Step 2, choose the appropriate field type.

**Field Types:**

| Type | Use When | Options? |
|------|----------|----------|
| `select` | One choice from a known list | Yes — 3-5 option .md files |
| `multiSelect` | Multiple choices from a known list | Yes — 4-8 option .md files |
| `text` | User provides a unique string value | No |
| `number` | User provides a numeric value | No |

**Decision guide:**
- Can you enumerate all possible values? → `select` or `multiSelect`
- Is the value unique to each user? → `text`
- Is the value a quantity? → `number`
- Can only one value be true at a time? → `select`
- Can multiple values be true? → `multiSelect`

**Prefer `select` over `text`** when possible — select fields provide rich option content that shapes agent behavior. Text fields only inject the raw string.

For each field, record:
- Field type
- Number of options (for select/multiSelect)
- Option names and slugs (kebab-case)

Present as an updated table. Confirm before writing option content.
