# Test with Thing Assistant

The **Thing** assistant is your built-in workspace automation AI. Use it to test and iterate on your workspace.

## Open Thing
Click the **Thing** button in the top-right header. A sliding panel (C10) opens from the right. The button text changes to "Hide Thing".

## What to Test

### Run workspace status check
Type `status` in the Thing input to get an overview of your workspace structure:
- Number of agents, flows, knowledge areas
- Configuration validity
- Runtime field summary

### Test agent actions via JSON
You can trigger agent actions using JSON envelopes in the Thing panel:
```json
{
  "action": "upsertAgent",
  "name": "MyAgent",
  "description": "What this agent does"
}
```

### Use the Chat sandbox
On the **Agent Configuration** page, click the violet **Chat** floating button (bottom-right) to open the Runtime Conversation Preview. Test your agent with real messages to verify:
- Instructions are interpreted correctly
- Knowledge context is injected as expected
- Slash commands trigger the right flows

## Iterate and Refine
Based on test results, return to the Studio Dashboard and adjust:
- Agent instructions (Main Instructions textarea)
- Knowledge area content (edit files in the tree explorer)
- Attached flows and their steps
