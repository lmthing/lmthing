Query the user's current billing status via the get-usage cloud endpoint and interpret the results. The endpoint returns the customer's Stripe balance in cents.

Interpret the balance value:
- Negative balance = credit remaining (prepaid amount not yet consumed)
- Positive balance = amount owed (usage beyond prepaid credit)
- Zero balance = no credit and no outstanding charges

Explain the user's token consumption patterns. Each LLM request is automatically metered by Stripe — input and output tokens are counted separately, and pricing varies by model (premium models like Claude Opus and GPT-4o cost more per token than basic models like Claude Haiku and GPT-4o-mini).

If the user is on the free tier and has exhausted their budget, note their options: wait for the next billing period or upgrade to Pro for higher limits.

    "currentTier": { "type": "string", "description": "free, pro, or computer" },
    "usageStatus": { "type": "string", "description": "healthy (within limits), warning (approaching limits), or exhausted (at limit)" },
    "recommendation": { "type": "string", "description": "Suggested action based on the usage status, or null if no action needed" }
  },

}
