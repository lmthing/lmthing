# Monitor Health

Set up ongoing monitoring for the deployed space:

**Health checks** run automatically every 10 seconds via HTTP GET to /health with a 5-second timeout. A healthy space consistently responds with 200 status.

**Monitoring practices:**
- Check space status periodically via list-spaces or get-space
- Watch for status changes from "running" to "failed"
- Monitor response times — increasing latency may indicate resource issues
- Review logs for errors or warnings

**Cost management:**
- Running spaces accrue compute costs continuously
- Stop spaces that don't need to be always-on (dev spaces, demos between events)
- Use start-space/stop-space to control when the space is active
- Consider the total cost of all running spaces — it adds up

**When to take action:**
- Health checks failing → investigate immediately, consider restart
- High resource usage → review what's consuming resources
- Space idle for days → consider stopping to save costs
- Space no longer needed → delete to clean up all resources
