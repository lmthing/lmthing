# Define Your Subject

Before building anything, clarify what this space is about.

Ask the user these questions:

1. **What is the subject?** (e.g., "meal planning", "code review", "customer onboarding")
2. **Who will use it?** (e.g., developers, teachers, marketers)
3. **What are 2-3 core tasks** this space should help with?
4. **What does an expert in this subject know?** Think about terminology, decision trees, reference data, and user preferences.

Use the creator's experience level to calibrate depth:
- **Beginners**: Help them narrow a broad idea into something concrete
- **Intermediate**: Validate their concept and suggest refinements
- **Advanced**: Quickly confirm the scope and move on

Output a one-paragraph **space brief** summarizing: subject, audience, and core use cases. This brief will guide all subsequent steps.
