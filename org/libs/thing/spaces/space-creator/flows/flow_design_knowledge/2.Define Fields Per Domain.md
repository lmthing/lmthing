# Define Fields Per Domain

For each domain identified in Step 1, plan the fields that will live inside it.

**Goal:** Each domain should have 1-3 fields. Each field represents one dimension of that domain's knowledge.

For each field, define:
- **Folder name** — kebab-case (e.g., `difficulty-level`)
- **Label** — Human-readable (e.g., "Difficulty Level")
- **Description** — What this field captures
- **Variable name** — camelCase version of folder name (e.g., `difficultyLevel`)

**Guidelines:**
- One field per decision the agent needs to make within this domain
- Fields should be orthogonal — selecting one field's value shouldn't determine another's
- If a domain only needs one field, that's fine — don't add fields for completeness
- The variable name must be unique across all domains in the space

Present all fields grouped by domain. Validate naming:
- Folder names: kebab-case
- Variable names: camelCase (exact conversion of the folder name)

Confirm before proceeding.
