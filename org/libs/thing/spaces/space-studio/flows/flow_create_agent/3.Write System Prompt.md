# Write System Prompt

Choose a prompt strategy and write the instruct.md with proper YAML frontmatter:

**Choose your strategy:**
- **Identity framing** (default) — Lead with role, communication style, expertise, do/don't list
- **Constraint setting** — Lead with rules: always/never/format/criteria. Best for reviewers
- **Knowledge referencing** — Structure around explicit domain references. Best for knowledge-heavy agents

**Write the instruct.md:**
1. YAML frontmatter: name (PascalCase), description, tools array, enabledKnowledgeFields array
2. Slash action tags linking to flows: `<slash_action name="..." description="..." flowId="flow_...">`
3. System prompt body following the chosen pattern

Keep the prompt concise but specific. Vague instructions like "be helpful" don't shape behavior. Specific instructions like "always show the formula first, then explain how it works" do.
