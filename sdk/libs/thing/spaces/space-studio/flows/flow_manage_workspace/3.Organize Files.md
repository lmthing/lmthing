Scan the workspace for naming convention violations and generate a reorganization plan with specific rename/move operations.

Check all files and folders against lmthing naming conventions:
- **Folders** must be kebab-case (agent-formula-expert, knowledge-design)
- **Agent names** in instruct.md frontmatter must be PascalCase (FormulaExpert)
- **Flow IDs** must use flow_snake_case for folder names (flow_create_space)
- **Flow steps** must be numbered Title Case files (1.Define Your Subject.md)
- **Variables** in field config.json must be camelCase (experienceLevel)
- **Domain references** in enabledKnowledgeFields must use the domain- prefix (domain-space-structure)

For each violation found, specify the current path/value, the corrected path/value, and the type of violation. Flag any renames that would break cross-references in other files and include those dependent updates.
