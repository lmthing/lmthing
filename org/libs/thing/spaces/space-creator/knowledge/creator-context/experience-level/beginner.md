# Beginner

The user is new to creating lmthing spaces. They may be familiar with AI tools generally but haven't built a space before.

## Characteristics

- Unfamiliar with the space file structure (agents, flows, knowledge hierarchy)
- May not know JSON or YAML syntax — needs copy-paste-ready examples
- Benefits from seeing complete examples before attempting their own
- Prefers guided, step-by-step processes over open-ended advice
- Likely to make naming convention mistakes (wrong casing, missing prefixes)

## How to Adapt

- **Explain concepts before using them** — Don't assume they know what a "domain" or "field" is
- **Provide complete file contents** — Show the full JSON/markdown, not just snippets
- **Validate as you go** — After each step, confirm the output looks correct before moving on
- **Use the simplest patterns** — Recommend Who-What-How for domains, Specialist for agents, Identity-First for prompts
- **Keep scope small** — Suggest 2-3 domains, 2 agents, and 1 flow for a first space
- **Warn about common mistakes** — Missing `domain-` prefix, wrong casing, mismatched flowIds

## What They Need Most

Clear, sequential instructions with zero ambiguity. Every file they need to create should be shown in full, with an explanation of what each part does and why it's there.
