# Apply Fix

Execute the appropriate fix based on the diagnosis:

- **Restart machine** — Stop then start via the dashboard or API. Resolves most transient issues. Volume data is preserved.
- **Clear disk space** — Remove temp files (`/tmp/*`), old logs, unused packages (`npm cache clean --force`). Target the largest consumers.
- **Kill runaway processes** — Identify with `top`, kill with `kill <pid>` or `killall <process>`. Prevents resource exhaustion.
- **Upgrade subscription** — If subscription expired, renew via billing portal. Machine access requires active Computer tier.
- **Refresh tokens** — If terminal disconnects due to token expiry, the dashboard should handle this automatically. Manual: call issue-computer-token.
- **Reprovision** — Last resort. Destroy and reprovision the machine if the issue is fundamental. Push to GitHub first to preserve workspace data.

After applying the fix, verify the issue is resolved by re-checking the original symptoms. If the fix didn't work, return to the diagnosis step and look for a different root cause.
