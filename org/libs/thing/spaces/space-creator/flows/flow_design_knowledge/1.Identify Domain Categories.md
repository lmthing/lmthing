Analyze the space's subject and map it into top-level knowledge domains. You must produce 3-6 domains that together provide comprehensive, non-overlapping coverage.

Evaluate which domain pattern best fits this subject:

- **Who-What-How** — Split by audience, subject matter, and methodology. Choose this when the agent needs to adapt to different types of users.
- **Input-Output** — Split by what goes in, what comes out, and transformation rules. Choose this for task-oriented spaces focused on converting inputs to outputs.
- **Layered Context** — Split from broad environment down to narrow specifics. Choose this for complex subjects with multiple interdependent layers.

For each domain, determine:
- A kebab-case folder name (e.g., `cooking-method`)
- A human-readable label (e.g., "Cooking Method")
- A single emoji icon representing the domain
- A hex color code for UI rendering
- A one-sentence purpose statement describing what knowledge lives here

You MUST include at least one domain that captures user context (role, experience, preferences) with at least one required field that the agent will prompt for at runtime.

Validate that:
- No two domains overlap significantly in scope
- Together the domains cover the subject comprehensively
- Domain names follow kebab-case convention
