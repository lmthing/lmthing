# Check Logs

Review machine logs, health check results, and system metrics:

- **System logs** — `journalctl -n 50` for recent system events, look for error messages and warnings
- **Health checks** — Access /health endpoint, check if it responds within the 5-second timeout
- **Process list** — `ps aux` to see all running processes, identify anything unexpected
- **Resource metrics** — `top` for CPU/memory, `df -h` for disk, `free -h` for memory detail
- **OOM events** — `dmesg | grep -i oom` to check for out-of-memory kills

Look for patterns: errors clustering at a specific time, resource usage spikes, repeated crash/restart cycles. Correlate log entries with the reported symptoms and any user-reported timeline.
