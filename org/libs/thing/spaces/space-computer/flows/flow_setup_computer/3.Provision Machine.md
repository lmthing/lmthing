# Provision Machine

Call the provision-computer endpoint with the chosen region. Explain that provisioning is asynchronous:

1. The API returns immediately with status "provisioning"
2. In the background: Fly.io app is created, volume is allocated, machine is launched
3. Health checks verify the machine is responsive
4. Status updates to "running" on success or "failed" on error

Provisioning typically takes 30-60 seconds. Monitor the status transition — if it stays in "provisioning" for more than 2 minutes, something may be wrong. If it fails, try a different region or check subscription status.

One computer per user is enforced — if they already have one, guide them to start it instead.
