# Choose Knowledge Domains

Identify which knowledge domains the agent needs access to:

- Map out `enabledKnowledgeFields` — each entry uses the `domain-` prefix followed by the kebab-case domain folder name
- Only attach domains the agent actually needs — avoid bloating the context with irrelevant knowledge
- Consider which fields should be `runtimeFields` — these must be populated by the user before the agent can run

For each domain, decide how the agent should use it. A knowledge-referencing prompt strategy works well when the agent has 3+ domains and needs specific per-domain instructions.

Check that the domains you reference actually exist in the space's knowledge/ directory. Missing domain references cause runtime errors.
