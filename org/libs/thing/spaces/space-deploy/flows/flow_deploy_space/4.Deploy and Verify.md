# Deploy and Verify

Create the space deployment and verify it's running:

1. Call create-space with `{ name, slug, description, region }`
2. The API returns immediately with status "provisioning"
3. Monitor the status: poll via get-space until status changes
4. Provisioning creates: Fly.io app → 1 GB volume → machine → health checks
5. Expect 30-60 seconds for provisioning to complete

**Verification checklist:**
- Status transitioned to "running"
- Space is accessible at `{fly_app_name}.fly.dev`
- Health endpoint responds at `/health`
- If auth_enabled, the SSO redirect works correctly
- If custom domain is set, the domain resolves correctly

If provisioning fails, check the status for error details. Try a different region if the issue is capacity-related. Verify the slug is valid (lowercase alphanumeric with hyphens).
