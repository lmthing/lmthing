Gather detailed information about the issue by investigating these symptom categories:

- **Reachability** — Is the machine reachable? Does the dashboard load? Does the health endpoint respond?
- **Performance** — Are commands slow? Is the terminal laggy? Are operations timing out?
- **Failures** — Which commands fail? What error messages appear? Which commands still work?
- **Connectivity** — Is the terminal disconnecting? How frequently? Does reconnecting work?
- **Recent changes** — New packages installed? Heavy processes running? Configuration changes?

Classify the symptoms by severity and identify patterns. Multiple symptoms often point to a single underlying issue — resource exhaustion, for example, causes slowness, command failures, and disconnections simultaneously.
