Based on the diagnosis, select and execute the appropriate fix:

- **Restart machine** — Stop then start via dashboard or API. Resolves most transient issues. Volume data is preserved.
- **Clear disk space** — Remove temp files (/tmp/*), old logs, unused packages (npm cache clean --force). Target the largest consumers.
- **Kill runaway processes** — Identify with `top`, kill with `kill <pid>` or `killall <process>`. Prevents resource exhaustion.
- **Upgrade subscription** — If subscription expired, renew via billing portal. Machine access requires active Computer tier.
- **Refresh tokens** — If terminal disconnects due to token expiry, dashboard should handle this automatically. Manual: call issue-computer-token.
- **Reprovision** — Last resort. Destroy and reprovision the machine if the issue is fundamental. Push to GitHub first to preserve workspace data.

After applying the fix, verify the issue is resolved by rechecking the original symptoms. If the fix did not work, return to the diagnosis step and investigate alternative causes.
