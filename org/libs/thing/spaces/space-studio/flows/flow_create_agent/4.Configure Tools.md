# Configure Tools

Decide which tools the agent needs and wire up the configuration:

**Available tools:** read, edit, search

- **read** — For agents that examine files. Most agents need this.
- **edit** — For agents that modify files. Only for agents that actively change workspace content.
- **search** — For agents that find files by name or content. Useful for large workspaces.

**Configure config.json:**
- Set `runtimeFields` — map domain names to field slugs that must be populated before the agent runs
- Keep runtimeFields minimal — only require fields the agent genuinely cannot function without

**Add slash actions:**
- Link flows to the agent via `<slash_action>` tags in instruct.md
- The `flowId` must exactly match the flow folder name (flow_snake_case)
- Each slash action needs a name, description, and flowId

**Create values.json** as empty `{}` — it's populated at runtime with user selections.
