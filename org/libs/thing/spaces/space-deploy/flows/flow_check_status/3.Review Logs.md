# Review Logs

Examine machine logs for errors, warnings, or unusual patterns:

**What to look for:**
- Error messages and stack traces — indicate application-level issues
- Out-of-memory (OOM) events — the machine exceeded its 1 GB RAM limit
- Crash loops — the process keeps crashing and restarting
- Connection failures — issues reaching external services
- Health check failures — logged when /health doesn't respond in time

**Log sources:**
- Machine logs via Fly.io dashboard or API
- Application logs within the container
- Health check results from Fly.io's monitoring

Correlate log entries with the reported status and any timing patterns. Issues that occur at specific times may be triggered by scheduled tasks, traffic spikes, or external service outages.
