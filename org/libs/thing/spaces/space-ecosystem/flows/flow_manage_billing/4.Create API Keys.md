Determine the user's API key management need and generate the appropriate action. Handle three operations:

**Creating a key:**
- Call create-api-key endpoint (accepts optional `name` parameter)
- Returns the raw key once: `lmt_` followed by 40 hex characters
- The key is NEVER shown again — the backend stores only the SHA-256 hash
- Instruct the user to store it securely immediately

**Listing keys:**
- Call list-api-keys to see all keys with prefix, name, created_at, revoked_at
- Only the first 12 characters (prefix) are returned for identification — never the full key

**Revoking keys:**
- Call revoke-api-key with the key_id to soft-delete
- Revoked keys immediately stop working
- Any key that may have been exposed should be revoked

Also explain usage: include the key in the Authorization header as `Bearer lmt_xxxxx...`. Works with all authenticated cloud endpoints. Same capabilities as JWT auth — both resolve to the same user identity.
