# Create Agents

Design and create the agents that will live in this space.

**Goal:** Define 2-3 agents with distinct roles that cover the space's core use cases.

For each agent, create three files:

### instruct.md

Write the frontmatter:

- **name** — PascalCase (e.g., `MealPlanner`)
- **description** — One sentence on the agent's expertise
- **tools** — Array of tool names (e.g., `["read", "search", "edit"]`)
- **enabledKnowledgeFields** — Domains to attach, with `domain-` prefix

Add slash actions for any flows this agent should trigger:

```xml
<slash_action name="Action Name" description="What it does" flowId="flow_action_name">
/command
</slash_action>
```

Write the main instructions using the appropriate **Prompt Strategy**:

- **Identity-First** — Default for specialists
- **Constraint-Driven** — For reviewers and validators
- **Knowledge-Referencing** — For agents with many domains

### config.json

Define `runtimeFields` — which fields should prompt the user at conversation start.

### values.json

Always starts as `{}`.

Validate all references:

- `enabledKnowledgeFields` values match knowledge folder names with `domain-` prefix
- `flowId` values match flow folder names
- `runtimeFields` keys match domain and field folder names
