# Review Workspace Structure

Examine the current workspace file tree and assess its organization:

- List all spaces in the workspace — each should have a package.json with name and version
- Within each space, check for the three pillars: agents/, flows/, knowledge/
- Identify orphaned files that don't belong to any space structure
- Check for missing configurations (agents without config.json, flows without index.md)
- Note any empty directories or placeholder files that should be filled

Build a mental map of the workspace before making changes. Understanding the current state prevents accidental file conflicts and helps prioritize organization work.
