# Connect Services Together

Explain how data flows between lmthing services so the user understands the bigger picture:

- **Workspace sync** — All workspace data (agents, flows, knowledge) lives in a private GitHub repo. Studio, Computer, and Space all sync from this repo. Changes in one place are available everywhere after a push/pull.
- **Authentication** — All services authenticate through com/ via SSO. One login works everywhere. API keys (lmt_) provide programmatic access.
- **LLM proxy** — All AI requests go through the generate-ai edge function → Stripe LLM proxy → provider. Usage is metered automatically regardless of which service initiated the request.
- **Deployment pipeline** — Build in Studio → deploy via Space → list on Store. Or build in Studio → run on Computer for personal use.

Help the user see lmthing as an integrated platform, not a collection of separate tools. Each service has a clear role in the overall workflow.
