# Set Access Controls

Configure who can access the deployed space:

**Authentication (auth_enabled):**
- Enable for private spaces with user-specific data or restricted access
- Disable for public tools, demos, and open documentation
- Can be changed anytime via update-space without redeploying

**Custom Domain:**
- Optional — set a custom domain for professional branding
- Requires DNS CNAME configuration pointing to the Fly.io hostname
- TLS certificates are managed automatically
- One custom domain per space, must be unique

**App Config:**
- Set any space-specific configuration as a JSONB object
- Feature flags, default agent, theme, environment overrides
- Remember: the entire object is replaced on update, not merged

Configure these settings based on the space's intended audience and use case. Public spaces for the store typically have auth disabled and may benefit from a custom domain.
