# Create API Keys

Guide the user through API key management for SDK and CLI access.

**Creating a key:**
- Call create-api-key endpoint (accepts optional `name` parameter)
- Returns the raw key once: `lmt_` followed by 40 hex characters
- The key is NEVER shown again — store it securely immediately
- The backend stores only the SHA-256 hash for verification

**Listing keys:**
- Call list-api-keys to see all keys with prefix, name, created_at, revoked_at
- The full key is never returned — only the first 12 characters (prefix) for identification

**Revoking keys:**
- Call revoke-api-key with the key_id to soft-delete
- Revoked keys immediately stop working
- Revoke any key that may have been exposed

**Using keys:**
- Include in the Authorization header: `Bearer lmt_xxxxx...`
- Works with all cloud endpoints that accept authentication
- Same capabilities as JWT auth — both resolve to the same user identity
