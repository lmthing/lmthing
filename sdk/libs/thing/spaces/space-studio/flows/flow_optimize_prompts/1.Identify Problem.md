Analyze the user's description of what is going wrong with their agent's conversations and produce a structured problem diagnosis.

Classify the issue into one or more categories:
- **Off-topic responses** — the agent answers questions outside its expertise
- **Too verbose** — responses are unnecessarily long with irrelevant information
- **Too generic** — responses lack specificity and could come from any LLM
- **Missing context** — the agent does not use its knowledge domains effectively
- **Wrong tool usage** — the agent uses tools incorrectly or at wrong times
- **Inconsistent tone** — the agent's communication style varies between responses
- **Hallucination** — the agent fabricates information not in its knowledge base

Ask for specific examples of problematic responses if the user has not provided them. Concrete cases are far more useful than vague complaints. Identify the primary symptom and any secondary symptoms that may share a root cause.
