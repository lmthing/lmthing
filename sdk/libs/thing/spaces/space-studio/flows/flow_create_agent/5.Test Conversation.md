Using the complete agent definition from previous steps, generate a set of test conversation scenarios and evaluate the agent's readiness.

Generate test cases covering:
- **In-role query** — a question squarely within the agent's expertise. Expected: confident, specific response using knowledge domains.
- **Out-of-scope query** — a question outside the agent's expertise. Expected: graceful decline referencing the correct agent.
- **Slash action trigger** — invoke each slash action. Expected: triggers the linked flow correctly.
- **Knowledge usage** — a query requiring information from the agent's knowledge domains. Expected: response incorporates domain context.
- **Tool usage** — if the agent has tools, a scenario requiring tool calls. Expected: appropriate tool selection and usage.
- **Edge case** — a query at the boundary of the agent's expertise. Expected: handles ambiguity without hallucinating.

For each test case, specify the input message, expected behavior, and what would indicate a problem. Flag any issues found and suggest specific instruct.md adjustments.
