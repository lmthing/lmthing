For each field defined in the previous step, determine the appropriate field type and enumerate options where applicable.

Field type selection guide:
- `select` — Use when there is a known list of mutually exclusive choices (3-5 options). This is the most common and preferred type because it provides rich option content that shapes agent behavior.
- `multiSelect` — Use when multiple values can be active simultaneously (4-8 options).
- `text` — Use only when the value is unique to each user and cannot be enumerated (e.g., a project name, a custom URL).
- `number` — Use when the value is a quantity (e.g., team size, budget).

Decision framework:
1. Can you enumerate all possible values? If yes, use `select` or `multiSelect`.
2. Can only one value be true at a time? If yes, use `select`. If multiple can be true, use `multiSelect`.
3. Is the value unique to each user? Use `text`.
4. Is the value a quantity? Use `number`.

Always prefer `select` over `text` when possible. Select fields inject rich, structured option content into the agent's context. Text fields only inject the raw string.

For each select/multiSelect field, list the option names and their kebab-case slugs. Options should be distinct, comprehensive, and ordered logically.
