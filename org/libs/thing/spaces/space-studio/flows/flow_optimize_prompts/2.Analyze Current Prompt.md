# Analyze Current Prompt

Read the agent's instruct.md and check for common issues:

- **Prompt strategy** — Is there a clear strategy (identity, constraint, knowledge-referencing)? Or is the prompt unstructured?
- **Role clarity** — Is the agent's identity and expertise clearly defined? Vague roles lead to vague responses.
- **Boundaries** — Is there a "do NOT" section? Missing boundaries cause role drift.
- **Knowledge wiring** — Are enabledKnowledgeFields appropriate? Too many bloats context; too few leaves gaps.
- **Tool guidance** — If tools are enabled, does the prompt explain when and how to use them?
- **Model match** — Is the selected model suited to the prompt style? (Claude for nuance, GPT for tools)
- **Temperature** — Is it appropriate? Lower (0.3-0.5) for precision, higher (0.7-1.0) for creativity.

Identify the root cause before suggesting fixes. Often the problem is structural (wrong strategy) rather than cosmetic (word choice).
