Using the original problem diagnosis and the applied modifications from previous steps, generate a structured test plan to validate the improvements.

For each modification applied:
- Reproduce the exact problematic prompts that triggered the original issue
- Test edge cases at the boundary of the agent's expertise
- Verify the fix did not introduce regression (e.g., over-constraining made the agent refuse valid questions)
- Check that knowledge domains are being used appropriately after changes
- Verify tool usage patterns are correct after any tool guidance updates

Compare before and after behavior on the same inputs. Each test should produce a clear pass/fail result. If a test fails, identify whether to revert the change, adjust it, or return to the analysis step for a different root cause.

Prompt optimization is iterative — expect 2-3 rounds for complex agents. Document what changed and why for each round.
