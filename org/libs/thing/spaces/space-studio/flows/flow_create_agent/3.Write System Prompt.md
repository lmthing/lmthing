Using the agent role definition and knowledge configuration from previous steps, generate a complete instruct.md file for the agent.

Select and apply the most appropriate prompt strategy:
- **Identity framing** (default) — lead with role, communication style, expertise, do/don't list. Best for most specialists.
- **Constraint setting** — lead with rules: always/never/format/criteria. Best for reviewers.
- **Knowledge referencing** — structure around explicit domain references. Best for agents with 3+ domains needing per-domain instructions.

The instruct.md must include:
1. YAML frontmatter with: name (PascalCase), description, tools array, enabledKnowledgeFields array
2. Slash action tags linking to flows: `<slash_action name="..." description="..." flowId="flow_...">` with the command on a line inside the tag
3. System prompt body following the chosen strategy pattern

Write specific, actionable instructions. Avoid vague directives like "be helpful." Instead write concrete instructions like "always show the formula first, then explain how it works." Include a clear "You do NOT" section to establish boundaries.
