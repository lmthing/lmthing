# Set Defaults and Requirements

Finalize each field's configuration by setting defaults and requirements.

**For each field, decide:**

### Default Value
- Which option should be pre-selected when the user doesn't make a choice?
- Choose the most common or safest option
- The default value must exactly match an option file's slug (kebab-case filename without `.md`)
- For text fields, default to empty string `""`
- For number fields, set a sensible starting value

### Required Flag
- `required: true` — The agent will prompt the user before starting. Use sparingly.
- `required: false` — The default value is used silently. Preferred for most fields.

**When to require a field:**
- The agent's behavior is fundamentally different based on the selection
- There's no safe default (e.g., the user's role in a consulting space)
- Getting the wrong value would produce unhelpful or misleading output

**When NOT to require:**
- A sensible default covers 80% of users
- The field is nice-to-have context, not essential
- Asking too many questions before starting creates friction

### Final Config Check

For each field, generate the complete `config.json`:
```json
{
  "label": "Field Label",
  "description": "What this field captures",
  "fieldType": "select",
  "required": false,
  "default": "option-slug",
  "variableName": "fieldVariable",
  "renderAs": "field"
}
```

Verify that every `default` value matches an actual option file slug in that field's folder.
