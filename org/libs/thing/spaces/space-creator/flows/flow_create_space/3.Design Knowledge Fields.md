# Design Knowledge Fields

For each domain from Step 2, design the fields.

**Goal:** Define 1-3 fields per domain with appropriate types and options.

For each field, specify:
- **Folder name** (kebab-case, e.g., `experience-level`)
- **Label** (human-readable)
- **Field type** — Reference the **Knowledge Design** domain:
  - `select` — One choice from a list (most common)
  - `multiSelect` — Multiple choices
  - `text` — Free-form user input
  - `number` — Numeric value
- **Variable name** (camelCase, e.g., `experienceLevel`)
- **Required** — `true` only if the agent can't function without it
- **Default value** — The most common/safe choice
- **Options** (for select/multiSelect) — List 3-5 option names

Validate all names against **Naming Rules**:
- Field folders: kebab-case
- Variable names: camelCase
- Option slugs: kebab-case

Present the field designs as a table grouped by domain. Confirm before proceeding.
