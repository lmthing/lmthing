# Documentation

Good documentation reduces support burden, increases user satisfaction, and leads to better reviews. Users should be able to get value from your agent without contacting you.

## Essential Sections

1. **Getting Started** — How to set up the agent after download. What runtimeFields to configure, what knowledge options to select, and how to start the first conversation.

2. **Configuration** — Explain each runtimeField: what it does, why it matters, and recommended values for common use cases. Include the variableName and expected input.

3. **Use Cases** — Specific scenarios with example prompts and expected outputs. Show 3-5 real-world situations where the agent excels.

4. **Limitations** — What the agent can't do. Being upfront about limitations builds trust and prevents frustrated users from leaving bad reviews.

5. **Troubleshooting** — Common issues and fixes. "If the agent gives generic responses, make sure you've selected the right knowledge options."

## For API-Access Agents

Include additional sections:

- **API Reference** — Endpoints, request format, response format
- **Authentication** — How to obtain and use access tokens
- **Rate Limits** — Any usage constraints
- **Error Handling** — Common error codes and what they mean

## Documentation Quality Checklist

- Can a new user get the agent working in under 5 minutes?
- Are all runtimeFields explained with recommended values?
- Are there at least 3 example prompts with expected outputs?
- Are known limitations clearly stated?
- Is the documentation scannable (headers, bullet points, short paragraphs)?

Well-documented agents get better reviews, fewer refund requests, and more word-of-mouth recommendations.
