# Plan Knowledge Domains

Using the space brief from Step 1, plan the knowledge domains.

**Goal:** Identify 3-5 top-level domains that together cover the space's subject comprehensively.

Reference the **Knowledge Design** domain for pattern guidance:
- **Who-What-How** — Best for spaces where the agent adapts to different audiences
- **Input-Output** — Best for transformation-focused spaces
- **Layered Context** — Best for complex, multi-layered subjects

For each planned domain, define:
- **Name** (kebab-case folder name, e.g., `user-context`)
- **Label** (human-readable, e.g., "User Context")
- **Icon** (single emoji)
- **Color** (hex code)
- **Purpose** (one sentence: what knowledge lives here)

Always include a context domain (like `creator-context` or `user-profile`) with at least one required field that the agent prompts for at runtime.

Present the domain plan as a table and confirm with the user before proceeding.
