For every select and multiSelect field, generate all option markdown files with rich, actionable content.

Each option file must contain:

**YAML frontmatter:**
- `title` — Human-readable option title
- `description` — One-line summary
- `order` — Sequential integer (1, 2, 3...) controlling display order

**Body content (150-300 words)** structured as:
1. **Definition** — What this option represents in clear, concrete terms
2. **Characteristics** — Key traits, behaviors, or properties that distinguish this option from its siblings
3. **Agent guidance** — Specific, actionable instructions for how the agent should modify its behavior when this option is selected
4. **Examples** (optional) — Concrete illustrations demonstrating this option in practice

Quality requirements for every option:
- **Actionable** — Selecting this option must produce a measurable change in agent behavior
- **Distinct** — Must be clearly different from other options in the same field
- **Self-contained** — Must be fully understandable without reading other options
- **Substantive** — At least 150 words of meaningful, domain-specific guidance

File naming: use the kebab-case slug as the filename (e.g., `beginner.md`). The slug must match what will be referenced as the `default` value in the field's config.json.

Order the options sequentially (order: 1, 2, 3...) in a logical progression (e.g., beginner before advanced, simple before complex).

Generate ALL option files. Do not abbreviate or skip any.
