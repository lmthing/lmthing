# Create Knowledge Domains

In the Studio Dashboard, go to the Knowledge section and create your top-level knowledge areas.

**Guideline:** Create 3–4 domains that together cover your subject comprehensively.

For each domain, click **+ Create Knowledge** and provide:
- **Name** — A concise label (e.g., "Customer Profile")
- **Description** — What types of knowledge live here (e.g., "Demographics, preferences, and support history")

After creation, click into each domain to open the Knowledge Area Details page. There you will:

1. Create **folders** for each sub-field using the "+ New Folder" button
2. Create **option files** (`.md`) inside each folder using "+ New File"
3. Write content in each file using the markdown editor — include key characteristics, best practices, and decision guidance

**Good domain examples:**
- `user-profile` → preferences, experience-level, role
- `platform-config` → environment, feature-flags, deployment-mode
- `content-type` → format, tone, audience
- `workflow-context` → trigger, goal, constraints
