# Choose Agent to Publish

Identify which agent from your workspace should be published to the store. Review the agent's config.json, instruct.md, knowledge domains, and flows to ensure it's complete and well-tested.

Check that all cross-references are valid — flowIds match flow directories, enabledKnowledgeFields reference existing domains, and runtimeFields point to real fields. The agent should work reliably in conversations before you publish it.

Consider whether the agent stands alone or is part of a larger space. If it depends on other agents or shared knowledge that won't be included, it may not work well for buyers. Ensure the agent is self-contained or clearly document any dependencies.
