For each domain category from the previous step, define the fields that will live inside it. Each field represents one dimension or decision point within that domain's knowledge.

For each field, determine:
- **Folder name** — kebab-case (e.g., `difficulty-level`)
- **Label** — Human-readable (e.g., "Difficulty Level")
- **Description** — What this field captures and why it matters for agent behavior
- **Variable name** — camelCase conversion of the folder name (e.g., `difficultyLevel`)

Design principles:
- 1-3 fields per domain. Do not add fields for completeness; each must represent a genuine decision the agent needs.
- Fields must be orthogonal — selecting one field's value should not determine another field's value.
- Variable names must be unique across ALL domains in the space, not just within a single domain.
- The variable name must be the exact camelCase conversion of the folder name.

Validate all naming:
- Folder names: kebab-case (lowercase, hyphens)
- Variable names: camelCase (no hyphens, first word lowercase)
