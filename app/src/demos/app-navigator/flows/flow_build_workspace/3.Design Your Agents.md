# Design Your Agents

In the Studio Dashboard, go to the Agents section and click **+ Create Agent** for each agent you need.

**Guideline:** Create 2–3 agents with distinct, complementary roles.

For each agent:
1. **Name** — Use PascalCase (e.g., `ContentWriter`, `QAReviewer`)
2. **Description** — One clear sentence describing its expertise
3. **Main Instructions** — Write a system prompt that defines:
   - The agent's role and personality
   - Tone and communication style
   - What it knows (from attached knowledge)
   - What it should and shouldn't do

After creating, use the **knowledge pills** (the bar below the header) to attach the relevant knowledge domains. Active pills appear in violet with a checkmark.

**Tips:**
- Agents inherit context from selected knowledge domains — attach only what's relevant
- Keep instructions focused; avoid redundancy with knowledge content
- Reference domain knowledge with natural language (e.g., "Use the user's experience level to calibrate your response")
