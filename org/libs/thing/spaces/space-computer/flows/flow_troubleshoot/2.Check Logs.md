Collect and analyze diagnostic data from the machine. Review these sources:

- **System logs** — `journalctl -n 50` for recent system events, identify errors and warnings
- **Health checks** — Access /health endpoint, verify response within the 5-second timeout
- **Process list** — `ps aux` to see all running processes, identify unexpected or runaway processes
- **Resource metrics** — `top` for CPU/memory usage, `df -h` for disk usage, `free -h` for memory detail
- **OOM events** — `dmesg | grep -i oom` to check for out-of-memory kills

Identify patterns: errors clustering at a specific time, resource usage spikes, repeated crash/restart cycles. Correlate log entries with the reported symptoms and any timeline the user provided.
