You are producing an action plan based on the inventory, health checks, and log analysis from previous steps. For each space that needs attention, recommend specific actions.

Action categories by situation:

**Unhealthy running spaces:**
- Restart (stop-space then start-space) — preserves data, fresh machine boot
- If restart doesn't help, escalate to application-level investigation

**Idle running spaces:**
- Stop to save costs (stop-space) — volume preserved, no billing
- Recommend a schedule if the space is needed periodically

**Failed spaces:**
- Try reprovisioning in a different region
- Check configuration for errors
- Review logs for the specific failure cause

**Long-stopped spaces:**
- Assess if still needed — if not, recommend deletion to clean up resources
- If needed occasionally, keep stopped at no ongoing cost

**Destructive actions (delete):**
- Always require explicit user confirmation
- Warn that deletion destroys the machine, volume, and app permanently
- Recommend pushing important data to GitHub first
- Status changes to "destroyed" (soft delete in database)

Prioritize actions by severity and potential cost savings. Document expected outcomes for each recommended action.
