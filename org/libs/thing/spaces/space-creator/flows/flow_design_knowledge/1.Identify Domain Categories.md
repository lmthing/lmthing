# Identify Domain Categories

Start by mapping the space's subject into top-level knowledge domains.

Ask the user to describe their space's subject. Then identify 3-6 domains using one of these patterns:

**Who-What-How** — Split by audience, subject matter, and methodology. Best when the agent adapts to different users.

**Input-Output** — Split by what goes in, what comes out, and transformation rules. Best for task-oriented spaces.

**Layered Context** — Split from broad environment down to narrow specifics. Best for complex subjects.

For each domain, define:
- **Folder name** — kebab-case (e.g., `cooking-method`)
- **Label** — Human-readable (e.g., "Cooking Method")
- **Icon** — Single emoji that represents the domain
- **Color** — Hex color code for the UI
- **Purpose** — One sentence: what knowledge lives here

Present the domains as a table. Check that:
- Domains don't overlap significantly
- Together they cover the subject comprehensively
- At least one domain captures user context (role, experience, preferences)

Confirm the plan before proceeding.
