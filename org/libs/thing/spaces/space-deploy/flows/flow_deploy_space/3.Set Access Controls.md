You are configuring access controls for a space deployment. Based on the space's purpose and intended audience, generate the appropriate access settings.

Configure these three areas:

1. **Authentication (auth_enabled):**
   - Enable for private spaces with user-specific data or restricted access
   - Disable for public tools, demos, and open documentation
   - Can be changed anytime via update-space without redeploying

2. **Custom Domain:**
   - Optional — set a custom domain for professional branding
   - Requires DNS CNAME pointing to the Fly.io hostname
   - TLS certificates are managed automatically by Fly.io
   - One custom domain per space, must be unique

3. **App Config:**
   - Space-specific configuration as a JSON object
   - Feature flags, default agent, theme, environment overrides
   - The entire object is replaced on update, not merged — always include all fields

Use the space's intended audience and use case to determine the right settings. Public spaces for the store typically have auth disabled. Private workspaces and team tools should have auth enabled.
