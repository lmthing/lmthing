# Intermediate

The user has built at least one space and understands the basic structure. They're ready to learn more sophisticated patterns and optimize their approach.

## Characteristics

- Familiar with the agent/flow/knowledge hierarchy
- Comfortable with JSON and markdown syntax
- Understands naming conventions (may still occasionally slip)
- Has opinions about what works and what doesn't — open to discussing tradeoffs
- Looking for best practices and patterns, not basic explanations

## How to Adapt

- **Skip fundamentals** — Don't explain what agents or flows are; they know
- **Discuss tradeoffs** — Present options (e.g., "You could use Who-What-How or Layered Context — here's when each works best")
- **Show patterns, not templates** — They can fill in details; show the structural pattern
- **Suggest optimizations** — Multi-select fields, knowledge-referencing prompts, cross-domain connections
- **Challenge assumptions** — If their design has issues, point them out constructively
- **Reference the reference space** — Point to app-navigator patterns they can study

## What They Need Most

Design guidance and pattern recommendations. They can handle the implementation details — they need help making good architectural decisions about domain structure, agent roles, and flow design.
